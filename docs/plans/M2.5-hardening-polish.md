## 1. What this milestone closes

Five items, ordered by independence:

### Sub 1 — Tests flaky under aggregate timing pressure

The Wave A retro documented this: vitest's per-test default timeout is 5 s, and a handful of integration-flavored tests (runTurn cost aggregation, override persistence, register_npc) routinely cross that under suite load — first-test-in-file warm-up, vi.resetModules() cycles, etc. Pass cleanly when run in isolation.

Fix: bump the per-file timeout to 30 s in `vitest.config.ts`. Single-line config; same fix the retro called out as the workaround.

### Sub 2 — Canonicality mode runtime enforcement

`Campaign.settings.canonicality_mode` (set by Conductor in SZ) is `"full_cast" | "replaced_protagonist" | "npcs_only" | "inspired"` per ROADMAP §10.4. Today the value persists through handoff but no runtime path consumes it.

Fix:
- `WorldBuilder` (the agent that validates player assertions, e.g. "I'm Naruto's brother") reads `canonicality_mode` from the campaign and applies the §10.4 rules:
  - `full_cast`: REJECT blood-relation claims to named canon characters; loose connections OK.
  - `replaced_protagonist`: REJECT contradictions of major canon arcs; player IS the protagonist.
  - `npcs_only`: no canon restrictions; canon characters are background flavor.
  - `inspired`: no canon restrictions at all; tone/style only.
- The mode threads into `WorldBuilderInput` from the turn workflow (already loads campaign settings). System prompt fragment per mode.
- Tests: per-mode WB decisions on a representative assertion battery.

### Sub 3 — UI /play polish

Current `/campaigns/[id]/play/page.tsx` is 48 LOC. Functional but bare.

Fix:
- **Auto-scroll on new turns.** Current behavior: text appears below the fold on long narratives; player has to scroll manually.
- **"KA is composing…" indicator** during the TTFT gap (turn submitted → first text delta). Visible chrome instead of a frozen UI.
- **Character sheet sidebar.** Compact card showing name + concept + power_tier + abilities collapse list. Read-only at this milestone (in-fiction edits via WorldBuilder, per ROADMAP §10.8).
- **Turn counter / today's spend visible** (already exists in header but small; no change here, just verify).
- **Pure layout work** — no agent / data-layer changes. Same SSE protocol, same hooks (`useTurnStream`).

### Sub 4 — Memory inspection UI

KA can call `recall_scene`, `search_memory`, `get_context_block` to consult the campaign's memory layers. The player has zero visibility into what those layers actually hold.

Fix:
- New `/campaigns/[id]/memory` route (server component).
- Three tabs:
  - **Episodic** — last N turns with their summaries (Chronicler-written).
  - **Semantic** — `semantic_memories` ordered by heat desc; category facets.
  - **Context blocks** — `context_blocks` with status + version.
- Hook the existing entities/episodic tools — read-only Firestore queries, no new tools.
- Linked from the /play header ("memory" button).

## 4. Risks

- **UI sub-commits are CSS-heavy** — Tailwind + a few state hooks. Easy to misalign with the existing /campaigns header look. Mitigation: copy the existing `/account/spending` header pattern verbatim; minimum new design surface.
- **Memory route may surface PII/sensitive content** that lives in semantic_memories. Today the player IS the campaign owner so reading their own campaign's memory is fine. Add a clear ownerUid check (mirrors `authorizeCampaignAccess`).
- **WorldBuilder canonicality enforcement may regress existing tests** that assume permissive mode. Mitigation: default to `inspired` when unset (the most permissive — no breaking).
