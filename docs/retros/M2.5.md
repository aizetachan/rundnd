## 2. Architecture decisions worth keeping

### 2.1 Sub 2 — `canonicality_breach` is a FLAG, not a REJECT

The ROADMAP §10.4 wording said "REJECT" for full_cast violations. The v4 system prompt for WorldBuilder says — explicitly, in caps — that REJECT doesn't exist. The split: ROADMAP described the player-facing intent ("canon must be respected"); v4 chose the mechanism (FLAG, not gatekeep). This sub kept v4's mechanism and added the typed flag so the UI can render canonicality breaches distinctly from internal-consistency contradictions. Player still gets to commit; sidebar just says "this contradicts canon — pick deliberately."

The flag is only emitted under `full_cast` and `replaced_protagonist` modes. `npcs_only` and `inspired` don't generate it — there's no canon to breach. The `violated_mode` field on the flag enforces this at the type level.

### 2.2 Sub 3 — character sidebar is server-loaded, not hook-fetched

`/play/page.tsx` now does a parallel `Promise.all` for turns + characters. The character card is read-only and changes rarely (only via WorldBuilder updates that regenerate the doc). Hook-fetching it client-side would add a request, a loading state, and a re-render trigger for no payoff. Server-rendered = simpler.

When in-fiction edits happen mid-session and the sidebar goes stale, a page reload picks up the new shape. Hot-update via SSE is M3+ (the same SSE we already use for turns could carry character-sheet deltas; not worth wiring at M2.5).

### 2.3 Sub 3 — "KA composing…" detects pre-TTFT, not just streaming

`streaming = true && displayedLive = ""` is the gap. Once the first text delta lands, `displayedLive` is non-empty and we drop the indicator + render the streaming bubble. Three bouncing dots feel "alive" without being noisy. The cursor pulse takes over once text is flowing.

### 2.4 Sub 4 — Memory inspection page is read-only by design

In-fiction edits happen via WorldBuilder; the player doesn't get a "delete this memory" button at M2.5. Reasons:
- Trust violation: the system promised the player it would remember; offering a delete-memory button breaks that contract.
- Memory layer churn would invalidate `recall_scene` heuristics.
- The actual problem isn't usually "wrong memory" but "out-of-date framing" — which Chronicler updates naturally on the next material turn.

If a memory is genuinely wrong (Chronicler hallucinated a fact), that's a Chronicler bug — fix it there, not in the memory layer. Player can always /override in fiction to override a memory's effect on play.

### 2.5 Sub 4 — semantic memories grouped by category

The semantic tab groups by `category` (relationship / location_fact / lore / etc.) before sorting by heat. Two reasons:
- The eye scans relationships in one place, lore in another. Grouping makes "what does the system remember about Vicious" a single section.
- Heat is meaningful within a category, not across — a hot relationship fact and a hot location fact are different signals.

The page query already top-N's by heat for the whole list; in-page rendering re-buckets. Same data, scannable shape.

## 3. What surprised us

### 3.1 The sidebar broke responsively only on mobile

`<aside className="hidden md:block">` — sidebar shows on tablet+ only. Mobile play mode keeps the chat full-width because squeezing a sidebar onto a phone screen is hostile. The character sheet is still accessible (will live behind a hamburger / bottom-sheet at M3+ when responsive design gets a proper pass).

### 3.2 The flakiness gate closed without ANY test changes

Just bumping `testTimeout`. Three runs verde in a row, no individual test rewritten, no flaky-mark annotations needed. Means the prior flakiness was 100% timing pressure — not race conditions in test logic. Worth remembering for the next time someone says "we should rewrite these flaky tests": the right answer might be "the timeout is wrong."

## 6. Where the codebase stands now

A new player can:
1. Sign in.
2. Land on `/campaigns` empty.
3. Click "+ Start a new campaign", walk through the conductor at `/sz` for any IP they name (single-source or hybrid), with the conductor researching unknown IPs via Path B (Opus + web_search).
4. Get handed off to `/campaigns/[id]/play` with a real opening scene.
5. See their character sheet alongside the chat (sidebar).
6. Watch "KA composing…" before the narrative streams in.
7. Click "memory" in the header to inspect what the system remembers — episodic prose, semantic facts grouped by category, context blocks at the entity level.

What's missing for "complete game experience" at this point is genuinely the M5+ stuff (images, voice, animations) — image generation in particular. The text-only loop is finished. M3 (replay/export, custom profile authoring UI) would add new shape; M5+ (ProductionAgent) would add new media.

## 7. Tone

The shape was right; the polish was the work. Sub 1 closed an inconvenience that affected nobody but the dev experience. Sub 2 made a typed gap explicit instead of conflating two distinct concerns. Sub 3 + 4 are textbook UI polish — the agents and data layer didn't change at all, the surface just stopped being bare.

The next plan (M3 or M5+) opens a new design surface. This one finished what was already there.
