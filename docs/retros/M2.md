## 2. Architecture decisions worth remembering

### 2.1 Conductor on the thinking tier (Sonnet 4.6 by default)

Plan + early docblocks asserted "Opus 4.7 + extended thinking." Audit caught the drift — the runtime threads `tier_models.thinking` which defaults to Sonnet 4.6 (cost-down on 2026-04-23). User explicitly chose to keep the thinking tier and update docs rather than switch to creative tier (Opus). Saves ~$0.15–0.35 per Session Zero session. See `project_session_zero_conductor_tier.md`. Same drift exists on `outcome-judge` and `director` — repo-wide doc rot, not specific to SZ.

### 2.2 KA/SZ surface partition

The `session_zero` AidmToolLayer is a deliberate non-cognitive layer. KA mounts only the eight cognitive servers (`buildMcpServers`) and the conductor mounts only `aidm-session-zero` (`buildSessionZeroMcpServers`). The two never see each other's tools. This isolates SZ as a separate orchestration over the same campaign surface, not a pipeline-stage running before KA — which preserves the v4 vision's "KA is the orchestrator" inversion at gameplay time and "Conductor is the orchestrator" inversion at SZ time.

### 2.3 Conductor's 5 tools, propose-vs-commit shape

Five tools instead of more granular per-field commits. `propose_*` tools record intent into `conversation_history` without mutating draft state; `commit_field` is the only mutation surface. The player chooses; the conductor commits. Hard-requirements gate (`commit_field` returns the snapshot, `finalize_session_zero` validates it transactionally) means a SZ doc cannot reach `ready_for_handoff` without all five required fields. The conductor doesn't gate-keep the player; the schema does.

### 2.4 HandoffCompiler split: LLM synthesis + deterministic merge

OpeningStatePackage has ~15 fields. The compiler asks the LLM for the ~12 fields requiring judgment (opening_situation, opening_cast, hooks, tone_anchors, world_context); the runner merges in the deterministic ones (player_character ← character_draft, package_metadata, director_inputs.initial_dna ← profile.canonical_dna). Saves tokens, prevents the LLM from "improving" data the player already confirmed, and keeps `fellBack` detection clean (sentinel identity comparison + `satisfies` marker so a future deep-clone refactor breaks loud).

### 2.5 Persistence ordering caught at audit

A late-found bug from sub 3: `conversation_history` is an `arrayUnion` array, but tool-call entries (written DURING a conductor run by `_history.ts`) and framing entries (written AFTER the run by `appendConversationTurn`) interleave. Without sort-on-read, the conductor's next-turn rendering put tool calls BEFORE the user message that triggered them. Fix: anchor the user-msg `createdAt` before the stream opens + sort by `createdAt` in `loadSessionZero`. Same audit caught aborted-stream message loss — fixed by always-attempting `appendConversationTurn` after the run loop regardless of stream success.

### 2.6 `handoff_in_progress` middle phase + revert-on-failure

Sub 4 first draft skipped this state (`ready_for_handoff` → `complete` directly). Audit caught: the type's docblock claimed `handoff_in_progress` exists, and sub 5's resume flow needs to distinguish quiesced from mid-compile. Now the orchestrator writes `handoff_in_progress` before the LLM call and flips to `complete` inside the post-compile transaction. On compiler exception, the catch reverts to `ready_for_handoff` so the player isn't bricked.

### 2.7 Redo soft-deletes prior FIRST

`redoSessionZero` calls `ensureSessionZeroCampaign` AFTER soft-deleting the prior campaign. If the order were reversed, the find-or-create would return the still-active prior and the redo would no-op. Plus `supersedes` pointer on the new campaign for diagnostics.

## 3. Audit findings caught

Subagent audits (Opus on Anthropic, ~700 words each) ran on every sub-commit before push. Findings split:

- **Important (fixed before push):** persistence-ordering bug, abort drops user msg, `handoff_in_progress` enum-vs-runtime drift, conductor model-tier doc drift, /meta UI text, stale destructive-action confirm.
- **Nits (fixed inline):** comment scrub per CLAUDE.md "WHY not WHAT", `satisfies` marker for fallback detection, Zod input ergonomics on `commit_field`.
- **Deferred (acknowledged as documented limits):** non-transactional `ensureSessionZeroCampaign` race (CTA double-click → 2 campaigns), no transcript-length cap in HandoffCompiler input, duplicate `coerceDate` helper, hook duplication between `useTurnStream` and `useSessionZeroStream`.

Audit cadence cost ~5–10 min per commit. Caught two important bugs that would have shipped otherwise.

## 4. What surprised us

### 4.1 The conductor's tool-call entries had to interleave with framing entries

Initial design (sub 3) wrote framing user/conductor entries AFTER the conductor's run. Tool-call entries from `_history.ts` already landed during the run. Without explicit chronological ordering, the next turn's render put the conductor's response BEFORE the user message that caused it — because the framing entries were appended last. Fix wasn't "always sort timestamps" — it was "stamp the user-msg time BEFORE opening the stream so its createdAt predates the tool-call writes." A subtle ordering invariant the schema didn't enforce.

### 4.2 Vitest's module isolation isn't free

Tests under aggregate-suite timing pressure occasionally hit 5s timeouts on the first test in a file (the registry import has to re-execute every module-init under `vi.resetModules()`). Caught in sub 2; documented in memory as known flakiness. `--testTimeout 30000` is the workaround.

### 4.3 Server-action confirm UX needs more reset hooks than expected

Two-stage destructive confirms (start-over + abandon) needed: (1) cross-reset between buttons, (2) submit-resets-confirm, (3) cancel-resets-confirm. Audit caught (2) only — sending a chat message could leave a stale "confirm abandon?" in the header. Fixed in sub 5.

### 4.4 The Profile.id field isn't the slug

Profile.id is `al_1` (AniList numeric id); the slug we use as the Firestore doc id is `cowboy-bebop`. Tests asserting `profile_id === "cowboy-bebop"` failed. The OpeningStatePackage stores `package_metadata.profile_id = profile.id`, not the slug. Worth a follow-up: align the field naming or document the semantic mismatch.

## 7. Tone

The audit cadence was the load-bearing discipline. Every sub-commit caught at least one important finding before push; without the audits, sub 3's persistence ordering bug, sub 4's `handoff_in_progress` drift, and sub 5's stale-confirm UX would have shipped. The user's preference for thorough commits + audits over many tiny commits is what made this milestone shippable inside one human-day-of-work.

The end-to-end loop now exists. From here, content (more profiles, hybrid synthesis, research subagents) sharpens what the shape already does.
